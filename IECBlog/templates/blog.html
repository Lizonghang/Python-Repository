刚接触Linux的时候觉得root用户多好啊，啥都能做。后来才发现能做的东西多了未必是好事。第一次在使用root用户上遇到坑是 npm install 上，以root身份install会报错 “cannot run in wd”，当时费了好大劲查错，最后才知道是用户的问题，官方文档是这么写的👇。

If npm was invoked with root privileges, then it will change the uid to the user account or uid specified by the user config, which defaults to nobody. Set the unsafe-perm flag to run scripts with root privileges.

关于 unsafe-perm 以后再讨论，我当时是直接更换其他用户解决的问题。那本次的指令就从新建用户开始吧。

新建用户 add user 　adduser & useradd

  • adduser 　自动为创建的用户指定主目录，shell版本，创建时需要输入密码

  • useradd 　创建用户，但没有指定主目录，shell，密码

  • useradd -d 　创建用户，指定主目录

切换用户 switch user 　su

  • su toxni 　切换到用户toxni，但仍保持现在的工作环境，这之后，通过命令exit或logout，或者是快捷键Ctrl+D即可返回原用户身份，相当于临时换个身份

  • su -l toxni 　切换到用户toxni，-l为login，可以省略，亦即 su - toxni，相比于上一个，这个指令是开启全新的工作环境

删除用户 delete user 　userdel

  • userdel toxni 　删除用户toxni，但仍保持 /home/toxni 存在

  • userdel -r toxni 　删除用户toxni，同时删除 /home/toxni 内容

  • userdel -f toxni 　强制删除用户toxni，即使现在在登陆状态

密码操作 password 　passwd

  • passwd toxni 　修改用户toxni的密码

  • passwd -l & --lock toxni 　锁定用户toxni不可修改密码

  • passwd -u & --unlock toxni 　解锁

  • passwd -S & --status 　打印当前密码状态

  • passwd -i & --inactive = DAYS 　密码过期多少天之后该用户被禁用

密码时效操作　 chage

  • chage -m 5 toxni 　5天之内不能修改用户toxni的密码

  • chage -M 5 toxni 　5天之后用户toxni的密码失效

  • chage -d toxni 　打印密码最后一次修改日期

  • chage -E toxni 　打印密码失效日期

  • chage -l toxni 　打印账户时效状态

  • chage -l 5 toxni 　密码过去5天之后用户toxni账户被禁






Linux最常用的一些文件操作指令，博主一直是用哪个查哪个再记住，所以比较零散。经常想重命名来着，然后发现文件被删除了😳，因为总把 mv 打成 rm。

删除 remove 　rm

  • rm -d & --directory 　直接把欲删除的目录的硬连接数据删成0，删除该目录

  • rm -f & --force 　强制删除文件或目录

  • rm -i & --interactive 　删除既有文件或目录之前先询问用户

  • rm -r & -R & --recursive 　递归处理，将指定目录下的所有文件及子目录一并处理

  • rm -v & --verbose 　显示指令执行过程

删除空目录 Remove Directory 　rmdir

切换工作目录 Change Directory 　cd

  • cd ../ 　上层目录

  • cd !$ 　把上个命令的参数作为cd参数使用

打印当前目录清单 list 　ls

  • ls -a & -all 　打印当前目录所有文件，包括“.”开头的隐藏文件

  • ls -A 　同上，但不列出“.”(表示当前目录)和“..”(表示当前目录的父目录)

  • ls -l & ll 　列出文件名，权限，所有者，文件大小等信息

  • ls -1 　每行只列出一个文件

  • ls -h & –human-readable 　以容易理解的单位列出文件大小，例如 1K 234M 2G

  • ls -tl 　以文件修改时间排序，-l 用于显示详情

  • ls -Sl 　以文件大小从大到小排序，-l 用于显示详情

  • ls -rSl 　以文件大小从小到大排序，-l 用于显示详情，-r 用于反转排序

  • ls -R & -recursive 　同时列出所有子目录层

创建目录 make directory 　mkdir

  • mkdir -m & --mode 600 IEC 　创建名为IEC的权限为600的文件夹

  • mkdir -p UESTC/IEC 　直接创建 UESTC/IEC 两个父子文件夹

  • mkdir -v 　显示创建详情

显示当前路径 Print Working Directory 　pwd

移动或重命名文件move files 　mv

  • mv -b 　移动若需要覆盖则先备份

  • mv -f 　强制移动重复则覆盖

  • mv -i & --interactive 　重复则询问是否覆盖

  • mv -u 　 文件较新才覆盖，亦即遇到重复保留新的一个

  • rm -t ~/iec toxni lypeer lumiaxu 　将toxni lypeer lumiaxu三个文件移动到 ~/iec 目录内

  • rm 　重命名也就相当于移动路径

最近在 O'Reilly 官网看见一本Eric A.Meyer 的新书《CSS Floating》，详细讲了一下 float 属性相关。正好闲来无事，把它翻译一下放在博客上把。纯属英语六级水平的渣翻，还请各位看官多多包涵。


Chap 1.1 浮动简介 (Floating)


很长时间以来，浮动元素(floated elements)一直是我们网页布局模型的基础要素。（很大程度上是因为clear性质，后面我们会提到的。）但是float从来不能称作真正意义上的布局，它在布局中的的使用就像是在走一条不太好的捷径，虽然能很快达到目的，但是给整体布局的影响是很不好的。用float来布局就像用table来布局一样，臭名昭著，麻烦多多。但是在以前，这也是我们仅有能用的方法。

如果正确使用的话，float还是相当有趣和好用的。尤其是最近版本的float被赋予了shaping属性，它能够允许与不规则几何体排版在一起的内容流动排版。

你大概很了解浮动元素(floated elements)的大部分特性了。从Netscape 1.1开始，像这样声明就可以让图片浮动。 <img src="b3.gif" align="right" alt="b3"> 如此一来，图片浮动到右侧并且让它周边的内容（比方说文本内容）流动排版，包围图片。事实上，“floating” 这个名字来源于Netscape DevEdge page 的“HEML 2.0 延伸(Extensions to HTML 2.0)”，原文是这样的：

The additions to your ALIGN options need a lot of explanation. First, the values "left" and "right". Images with those alignments are an entirely new floating image type.

在排列(align)选项中有很多东西需要解释，首先就得说“left”和“right”两个值。被赋予这两个值的图片俨然已是一种新的类型了，即浮动图片(floating image)。

在过去，只有图片可以浮动。部分浏览器下，表格内也可以使用浮动。引用CSS则可以让你浮动任何元素，从图片到文本段落，再到表单。在CSS内，这些浮动的行为是受float属性控制的。



FLOAT属性
可选值： left | right | none | inherit
适用范围： 所有元素
是否继承：否
计算值*：指定

举例来说，要让图片浮动于左，则可以这样写：
<img src="b4.gif" style="float: left" alt="b4">

正如下图所示，图片浮动在浏览器窗口的左侧，并且它周围的文字流动环绕排版，这应该是你所期望的效果。

然而，浮动元素应用在CSS里面时，又会有一些有趣的问题出现。详见下一节。






计算值：

CSS属性的 计算值 (computed value) 由指定的值计算而来，例如，如一个元素的属性值为 font-size:16px 和 padding-top:2em, 则 padding-top 的计算值为 32px (字体大小的2倍)，详见 计算值wiki 。
